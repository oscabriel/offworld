## Three Ways to Use Offworld

<Tabs>
<TabItem label="CLI for Humans">
Run commands directly in your terminal with human-readable output:

```bash
ow pull tanstack/router
ow project init
ow list
```

Perfect for:

- Manual reference management
- Pulling new references for your stack
- Terminal-first workflows
  </TabItem>
<TabItem label="Skill for Agents">
After running `ow init`, agents can access references autonomously:

```bash
ow init  # Symlinks skill to all agents
```

Then prompt your AI agent:

```
Use the Offworld skill to check what references are available,
then help me implement routing with TanStack Router.
```

Perfect for:

- Autonomous reference discovery
- Multi-step AI workflows
- Consistent knowledge across sessions
  </TabItem>
<TabItem label="Project Integration">
Scan your dependencies and install matching references:

```bash
cd my-project
ow project init
```

Creates `.offworld/map.json` and updates your `AGENTS.md` with a reference table.

Perfect for:

- Project onboarding
- Team consistency
- Dependency-aware context
  </TabItem>
</Tabs>

## Works Where You Work

| Environment     | How It Works                                   |
| --------------- | ---------------------------------------------- |
| **Terminal**    | Run `ow` commands anywhere                     |
| **Claude Code** | Skill auto-loads from `~/.claude/skills/`      |
| **Cursor**      | Native skill integration with Composer         |
| **Amp**         | Skills directory at `~/.config/agents/skills/` |
| **OpenCode**    | Terminal-native with skill support             |
| **Any Agent**   | Symlink the skill or read references directly  |

## How It Works

```
ow project init
  → Scan manifests (package.json, Cargo.toml, etc.)
  → Resolve dependencies to repositories
  → Clone repositories locally
  → Generate AI-optimized references
  → Update the clone map
  → Agents route to the right context instantly
```
