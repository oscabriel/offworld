## Three Ways to Use Offworld

<CardGrid>
	<Card title="CLI for Humans" icon="terminal">
		Run commands directly in your terminal to manage references and skills. ```bash ow pull
		tanstack/router ow project init ```
	</Card>
	<Card title="Skill for Agents" icon="rocket">
		Install the skill so agents can access reference knowledge autonomously. ```bash ow init #
		Symlinks skill to all agents ```
	</Card>
	<Card title="Project Integration" icon="document">
		Scan your project's dependencies and inject relevant references into AGENTS.md. ```bash ow
		project init ```
	</Card>
</CardGrid>

## Why Offworld?

<CardGrid stagger>
	<Card title="AI-Native Design" icon="rocket">
		Built for coding agents. References are optimized for LLM consumption with the right level of
		detail.
	</Card>
	<Card title="One Skill, All Agents" icon="puzzle">
		Single skill installation works with Claude Code, Cursor, Amp, OpenCode, Codex, and Gemini.
	</Card>
	<Card title="Dependency-Aware" icon="magnifier">
		Scans your package.json, Cargo.toml, or go.mod and installs only the references you need.
	</Card>
	<Card title="Community References" icon="open-book">
		Browse and share references on offworld.sh. Pull from the community or push your own.
	</Card>
</CardGrid>
